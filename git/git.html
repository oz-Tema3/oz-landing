자주 묻는 질문 (Git FAQ)

Q. git pull과 git fetch는 뭐가 다른가요?

A. 간단히 말해, fetch는 원격 저장소의 최신 변경 내역을 가져오기만 하고(확인만 하는 단계), pull은 변경 내역을 가져와 현재 브랜치에 합치기(병합)까지 합니다.

  • git fetch: 원격 저장소의 최신 데이터를 내 로컬 저장소로 가져옵니다. 하지만 내 작업 파일에는 아무런 영향을 주지 않습니다. 팀원이 어떤 작업을 했는지 확인하고 싶을 때 안전하게 사용할 수 있습니다.
  • git pull: git fetch + git merge 입니다. 원격 저장소의 변경사항을 가져와서 현재 내 브랜치와 바로 병합합니다.
Q. commit을 되돌리고 싶어요. reset과 revert의 차이는 뭔가요?

A. 둘 다 작업을 되돌리는 기능이지만, 커밋 기록을 다루는 방식이 다릅니다. 협업 시에는 revert 사용이 권장됩니다.

  • git reset: 특정 커밋 시점으로 아예 돌아가 버립니다. 그 이후에 쌓았던 커밋 기록이 "사라집니다." 주로 개인 브랜치에서 아직 push 하지 않은 커밋을 정리할 때 사용합니다.
  • git revert: 특정 커밋의 변경 사항을 "취소하는 새로운 커밋"을 만듭니다. 기존 커밋 기록은 그대로 유지되기 때문에, 팀원들과 함께 사용하는 원격 브랜치의 작업을 안전하게 되돌릴 수 있습니다.
Q. 지금 하던 작업을 잠시 저장하고 다른 브랜치로 가고 싶어요.

A. 아직 커밋하기 애매한 작업들을 임시로 저장할 때 git stash를 사용합니다. 

# 현재 작업 내용 임시 저장
git stash

# 다른 브랜치에서 작업 후... 다시 돌아와서
# 가장 최근에 저장한 내용 가져오기
git stash pop

stash를 이용하면 작업 디렉토리가 깨끗해져서 다른 브랜치로 자유롭게 이동할 수 있고, 나중에 다시 돌아와 stash pop으로 저장했던 작업을 이어갈 수 있습니다.

Q. Merge Conflict (병합 충돌)는 왜 발생하고 어떻게 해결하나요?

A. 충돌은 여러 브랜치에서 같은 파일의 같은 줄을 수정했을 때 Git이 어떤 것을 선택해야 할지 몰라서 발생합니다.

해결 과정:

  1. git status 명령어로 어떤 파일에서 충돌이 났는지 확인합니다.
  2. 해당 파일을 열면 <<<<<<< HEAD, =======, >>>>>>> 표시가 보입니다.
  3. 이 표시들을 기준으로, 팀원과 상의하여 어떤 코드를 남길지 결정하고 파일을 직접 수정합니다. (표시자들도 모두 지워야 합니다)
  4. 수정이 완료된 파일을 다시 git add 하고 git commit 하면 충돌 해결이 완료됩니다.
